  1. lsコマンド

    2. git ls-files

    リポジトリ内のファイルをリスト表示

    git ls-files -c

    インデックにキャッシュされているファイルを表示

    git ls-files -m

    更新されたファイルを表示

    git ls-files -s

    インデックスのオブジェクト名、mode bits,stage number表示

    git ls-files -o

    マージされていないファイルの一覧を表示

    git ls-files　-u

    リポジトリ内のファイルをリスト表示

    git ls-files --exclude-standard -o

    .gitignoreにマッチするファイルを含めた追跡されていないファイルの一覧が表示

    git ls-files -v

    git管理下で無視したファイル(assume-unchanged,skip-workree)を確認する

    git stash list

    スタッシュに対比させたWIPのリスト表示

    git config --local --list

    ローカル（リポジトリ固有の）Git構成をリスト表示

    git config --global --list

    グローバル（ユーザ固有の）Git構成をリスト表示

    git config --system --list

    システム（サーバー全体の）Git構成をリスト表示

  2. echo役割・使い方

    3. 画面に文字列を表示する

    $ echo 表示するもの

    変数を表示する

    $ echo $変数名

    変数と文字列を同時に表示する

    $echo weight $a g

    変数を複数表示する

    $menu=coffee $price=500 $echo order:$menu price:$price Yen

    計算値を表示する

    $n=5 $echo Menu:$menu Number:$n price:'expr $price\* $n` Yen

    -eオプション：改行を表示する

    $echo -e １行目\\n２行目\\n3行目 \\n4行目

    -nオプション：出力文字の最後の改行をしない

    $echo -n 出力文字

    出力を画面ではなくファイルに出力する・新規作成したファイルに出力する

    $echo 文字列＞ファイル名

    $echo "h1{font-size:20px}"> style.css

    シングルクォーテーション『’』の場合

    中は全て文字列であると解釈される。

    ダブルクォーテーション『”』の場合

    変数が展開されて表示される

    バッククォーテーション『`』の場合

    引用府内をコマンドであるとみなして、その実行結果と置き換える

  3. catコマンド役割と使い方

    4. ファイルの閲覧

    $cat file名

    リダイレクト『＞』を用いることで、別のファイルに出力内容を記述することができる

    $cat fileA > fileB

    上書きでなく、追加がしたければリダイレクト『>>』を使用する

    $cat fileA >> fileB

    catコマンドによるファイルの結合

    catにはもともと「ファイルを連結して標準出力に出力する」というコマンド本来の意味合いがある。cat自体が、「連結する」を意味するconcatnateの略である。

    $ cat fileC fileD

    「cat -n ファイル名」により、行番号をつけて出力する。これは使うことが多いオプションなので覚えてしまおう。"number"のn

    $ cat -n fileB

    blankなし（空白行を入れずに）番号をつけて表示するオプション。"blank"のbで覚える。

    $ cat -b fileB

    連続した空白行を１行の空白行にまとめるオプション

    $ cat -s fileB

    各行の最後に"$"を表示するオプション

    $ cat -E fileB
